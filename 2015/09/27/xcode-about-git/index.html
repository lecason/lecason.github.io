Xcode Push 项目到 GitHub

Mac 10.10.5 (14F27)
Xcode Version 7.1 beta (7B60)

创建Repositories

在Github上创建Repositories

在Xcode中新建项目，勾选 Create Git repositories on My Mac

如果已经新建了的项目但没勾选Create Git repositories on My Mac的可以在Xcode - Source Control 中选择 Create Working Copy

选择 Xcode - Source Control - Project Name Master - Configure Project Name - Remotes - + - Add Remote

然后再在Source Control里面Push就可上传到Github上面去了

这时候Github上就可以看到已经Push上去的文件了

提交Commint

Commint的意思是先把代码保存在本地，如果要更新到服务器的话，就Push上去，我们在viewDidLoad里面print(“Hello”)然后再 Xcoode - Source Control - Commint ，这样子代码就保存好在本地，然后再点Xcoode - Source Control - Push,这时候代码就会上传到服务器。效果：

更新项目Pull

如果其他人修改了服务器上的文件，那么我们要得到服务器上最新的文件我们就要Pull回来，比如说我们直接在服务器上编辑ViewController.swift,在print(“Hello”)里面加上一个World。然后保存Commint Changes：

这时候我们在Xcode - Source Control - Pull,我们就能得到服务器上更新的代码：

从 GitHub 下载项目到本地

从Xcode - Source Control 里面选择 Check Out ，输入Github上的地址后再保存到本地
